# 生产就绪部署链路

## 定义

**生产就绪部署链路**指网站不仅能响应请求，还具备持续可用、可观测、可恢复、可安全演进的全套基础设施与运维能力。它与「最小可跑链路」（域名 + HTTPS + VPS + 反向代理 + 前后端）之间的差集，即传统裸机/VPS 部署中常被遗漏的环节。

## 为什么重要

最小链路可在数小时内跑通 demo，但缺少进程守护、备份、监控、备案、WAF 等环节时，生产环境面临：进程崩溃无自愈、磁盘被日志撑满、证书过期即宕、国内 80/443 未备案被封、单点故障无回滚路径等可预期故障。原文的核心论点是：**自主部署 VPS 的隐性成本正在于补齐这七层能力**。

## 七层能力模型

| 层次 | 典型遗漏 | 关联概念/实体 |
|------|---------|--------------|
| 网络与访问 | DNS、[[ICP备案]]、安全组、DDoS/WAF | [[Cloudflare]] |
| 运行时 | [[进程守护]]、环境一致性、[[健康检查与探活]]、logrotate | systemd, pm2 |
| 数据 | DB 拓扑、[[Redis]]、对象存储、备份与恢复演练 | OSS/S3 |
| 发布运维 | [[CI-CD]]、环境隔离、[[灰度发布与回滚]]、密钥管理 | Jenkins |
| 安全合规 | HTTPS 自动续期、SSH 加固、CVE 审计、操作审计 | [[Certbot]] |
| 性能扩展 | 负载均衡、HTTP/2、压缩、缓存策略、连接池监控 | [[Nginx]] |
| 业务体验 | 邮件、SEO、错误页、熔断降级 | SendGrid, Sentinel |

## 与其他概念的关系

- **最小可跑链路** — 生产就绪链路的子集；前者验证功能，后者验证运营。
- **[[传统 VPS 部署完整链路]]** — 本概念在维基中的综合展开与对照表。
- **容器化/Serverless** — 平台托管七层中的多数环节，降低自建成本但引入 vendor lock-in 与平台学习曲线。[[Vercel]] 将上四层（infra/运行时/网络/发布）平台化，数据/合规/深度运维仍外置（见 [[传统 VPS 与 Vercel 部署对照]]）。
