# ICP备案

## 定义

**ICP 备案**（Internet Content Provider 备案）是中国大陆对提供互联网信息服务的网站/应用实施的准入制度。域名解析到境内服务器并对外提供 Web 服务前，须通过接入商（云厂商/VPS 提供商）向省级通信管理局提交备案，取得备案号后方可正常开放 80/443 端口。

## 为什么重要

在 [[生产就绪部署链路]] 的「网络与访问层」中，[[ICP备案]] 与 DNS、防火墙并列，但是**面向国内用户的硬门槛**：未备案域名即使技术栈完整，运营商/云厂商也会阻断 80/443 访问。这是「买了域名和 VPS 但还没打通」类故障中极易被技术团队低估的环节。

## 机制与流程（概要）

1. 选择具备备案资质的接入商（阿里云、腾讯云等）。
2. 提交主体信息（个人/企业）、域名、服务器 IP。
3. 管局审核（通常数个工作日至数周）。
4. 备案通过后，在网站底部展示备案号并链接至工信部查询页。
5. 部分场景还需**公安备案**（网安备案），与 ICP 备案相互独立。

## 边界与反例

- **纯境外托管 + 境外用户**：若服务器与用户均在境外，不适用 ICP 要求，但境内访问可能慢或被间歇性干扰。
- **仅内网/非 80/443 端口**：不走公网 Web 入口的服务可能无需 ICP，但业务形态受限。
- **备案 ≠ 内容合规**：备案解决接入准入，内容违法仍会被关停。

## 与其他概念的关系

- [[生产就绪部署链路]] — ICP 是访问层前置条件。
- **[[Vercel]]** — 节点全在海外，**不提供** ICP 备案能力；国内正式业务需 VPS/国内云 + 备案，或 Vercel + 国内 CDN 混合（见 [[传统 VPS 与 Vercel 部署对照]]）。
- [[Cloudflare]] — 橙云代理 + 未备案境内源站组合可能触发合规与连通性问题，需按实际架构评估。
